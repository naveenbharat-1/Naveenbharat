## Findings

### [MEDIUM] [SEC] 4 buckets are public with open listing
**Where:** `storage.buckets` — likely `book-covers`, `content`, `avatars`, `course-materials`.
**Why:** Anyone can enumerate every filename via `storage.objects` SELECT. Filenames often leak user IDs / lesson IDs.
**Fix:** Narrow SELECT policy on `storage.objects` to `WHERE bucket_id = '<x>' AND (auth.role() = 'authenticated' OR object metadata is whitelisted)`, or flip the bucket private and serve via signed URLs. Needs per-bucket call — waiting on which buckets are intentionally public before touching.

### [MEDIUM] [AUTHZ] SECURITY DEFINER fns still callable by anon
**Where:** ~10 fns flagged by linter 0028. Likely leftovers like `get_platform_stats`, `has_role` overloads.
**Why:** `SECURITY DEFINER` fns bypass RLS by design; wider EXECUTE means anon can call them.
**Fix (SQL — surface for approval):**
```sql
-- One-shot: revoke anon EXECUTE on every SECURITY DEFINER fn in public,
-- then re-grant only the intentionally-public ones (currently: get_platform_stats).
DO $$
DECLARE r RECORD;
BEGIN
  FOR r IN
    SELECT n.nspname, p.proname, pg_get_function_identity_arguments(p.oid) AS args
    FROM pg_proc p JOIN pg_namespace n ON n.oid = p.pronamespace
    WHERE n.nspname = 'public' AND p.prosecdef = true
  LOOP
    EXECUTE format('REVOKE EXECUTE ON FUNCTION %I.%I(%s) FROM anon, PUBLIC',
                   r.nspname, r.proname, r.args);
  END LOOP;
END $$;
GRANT EXECUTE ON FUNCTION public.get_platform_stats() TO anon, authenticated;
```

### [LOW] [SEC] GraphQL schema discoverability
**Where:** ~95 tables/views visible via `pg_graphql` to anon or authenticated.
**Why:** Discoverability only — RLS still blocks row reads. Low-risk unless a schema name leaks intent.
**Fix:** `REVOKE SELECT ON <table> FROM anon` on tables that have zero anon policies. Batch script but needs eyeball — some (courses, hero_banners, subscription_plans) intentionally public.
