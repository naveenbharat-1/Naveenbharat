## New findings (this pass)

### [MEDIUM] [SEC] [#20] Storage MIME/size caps still open
**Where:** all 13 buckets, `allowed_mime_types = NULL`, `file_size_limit = NULL`.
**Symptom:** authenticated user (or compromised admin) can upload `evil.html` to `content` / `book-covers` / `notices` → stored XSS on `*.supabase.co`.
**Root:** buckets created before MIME/size feature; Lovable's `storage_update_bucket` tool only exposes the `public` flag.
**Fix (user, ~2 min):** Supabase Dashboard → Storage → each bucket → *Edit bucket*:
- `avatars`, `book-covers`, `comment-images` → MIME `image/jpeg,image/png,image/webp,image/gif`, size 5 MB.
- `notices`, `chat-attachments`, `content` → same + `application/pdf`, size 20 MB.

### [LOW] [UX] Hardcoded brand hex in `CommunityStrip`
**Where:** `src/components/Landing/CommunityStrip.tsx:32,42` — `bg-[#229ED9]` (Telegram), `bg-[#FF0000]` (YouTube).
**Symptom:** breaks the "no raw hex" rule; won't invert in dark mode.
**Fix:** promote to design tokens `--brand-telegram` / `--brand-youtube` in `src/index.css`, use `bg-brand-telegram` etc. Backlog.

### [LOW] [MAINT] `key={idx}` on 5 mapped lists
**Where:** `TimetableManager.tsx:87`, `AskDoubtSheet.tsx:284`, `Dashboard.tsx:225`, `Reports.tsx:239`, `AdminAnalytics.tsx:417`.
**Symptom:** wrong element reused if the list reorders/filters (rare on these read-only lists, so LOW).
**Fix:** use stable id where available; leave `idx` where the list is truly static (DAY_NAMES is fine).

### [LOW] [UX] 4 `fixed`/`sticky` elements without safe-area
**Where:** `Downloads.tsx:338` (modal overlay — full-screen, fine), `AutoScrollFab.tsx:102` (sheet backdrop), `Library.tsx:247` (FAB right-5, no bottom offset), `LectureListing.tsx:420` (`top-[60px]` sticky — fine on web, may collide with status bar in native).
**Fix:** add `bottom-[calc(1.25rem+env(safe-area-inset-bottom))]` to Library FAB; leave overlays alone.
