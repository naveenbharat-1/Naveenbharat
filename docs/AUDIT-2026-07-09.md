## Findings

### [HIGH] [SEC] Supabase auth token read directly from `localStorage`
**Where:** `src/lib/pdfViewerUrl.ts:35` — `localStorage.getItem(\`sb-${SUPABASE_REF}-auth-token\`)`.
**Symptom:** Any XSS on the origin lifts a valid JWT + refresh token from the pdf viewer bootstrap path.
**Root:** Bypasses the supabase-js client; direct storage read couples us to internal key format and defeats future httpOnly/cookie migration.
**Fix (surface, not auto-applied):** use `supabase.auth.getSession()` and pass `session.access_token` into the viewer URL builder. Same file, small change — but touches auth, so awaiting approval per rules.

### [MEDIUM] [SEC] 109 Supabase linter warnings
**Where:** `supabase--linter` snapshot (2026-07-09).
**Symptom:** 3× public storage buckets allow listing; ~20× tables exposed to `anon` in the GraphQL/PostgREST schema that likely shouldn't be pre-login-discoverable; remaining are `function_search_path_mutable` and `extension_in_public` style advisories.
**Root:** Historical `GRANT SELECT ... TO anon` on admin/marketing/config tables that only need authenticated reads; buckets created with `public = true` + broad SELECT policy.
**Fix (surface):** batch migration — `REVOKE SELECT ... FROM anon` on admin/marketing/audit tables; tighten bucket SELECT policies to `owner = auth.uid()` or drop `list` capability. Needs review of each table's intended audience against `references/schema-map.md` before I ship SQL.

### [MEDIUM] [PERF] Fonts loaded from Google Fonts on critical path
**Where:** `index.html:56-58` — Inter+Fraunces+JetBrains Mono via `fonts.googleapis.com`.
**Symptom:** Two extra DNS+TLS handshakes before first meaningful paint on cold mobile; already softened by `media="print" onload="this.media='all'"` but font swap still visible.
**Fix:** self-host WOFF2 subsets in `public/fonts/` and `<link rel="preload" as="font" crossorigin>` the two critical faces.

### [LOW] [OBS] `localStorage("nb_admin_eruda")` gate is fine but undocumented
**Where:** `src/main.tsx:33`.
**Fix:** add a one-liner comment noting this is a dev-only debugger toggle so future audits don't flag it as an admin backdoor.

### [LOW] [UX] Boot screen forces cream (#F7F4EE) in dark mode
