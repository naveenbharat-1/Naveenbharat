## Findings (all resolved this pass)

### [MEDIUM] [PERF] pdf-proxy first-tap latency — FIXED
**Where:** `supabase/functions/pdf-proxy/index.ts`, `.github/workflows/pdf-proxy-keepalive.yml`
**Why it matters:** Edge function shutdown logs in this turn show frequent boots (`booted (time: 23ms)`) interleaved with `shutdown` events — every cold path adds ~1.5–3 s to first chip-open. Students perceive this as the "PDF stuck at 71 %" bug.
**Fix:** GitHub Actions cron pings the function every 10 min with a HEAD on a real allow-listed jsDelivr PDF. Free, no Drive quota cost, exercises the real code path so a deploy regression also fails the workflow.

### [MEDIUM] [PERF] Bundle audit — FIXED
**Where:** build output above
**Result:** Inside budget on every chunk. No new action needed. `eruda` (146 KB gz) and `vendor-sentry` (148 KB gz) are already lazy / dev-gated. `CartesianChart` (94 KB gz, recharts) is only on admin analytics, route-split.
**Follow-up (LOW, deferred):** `index.nohighlight` 84 KB gz — refractor stub already replaces `react-syntax-highlighter`. Further trimming would mean dropping markdown highlighting entirely; not worth it.

### [LOW] [UX] Hardcoded sticky offset — FIXED
**Where:** `src/pages/MyCourseDetail.tsx:107–127, 1000`
**Fix:** `breadcrumbBarRef` + `ResizeObserver` writes `document.documentElement.style.setProperty('--app-breadcrumb-h', `${h}px`)`. Secondary sticky bar uses `top-[var(--app-breadcrumb-h,44px)]`. If breadcrumb height ever changes (longer title wraps, smaller font on mobile, Hindi locale taller line-box), the offset adapts in the next frame. Cleanup on unmount removes the var.

## Loopholes / drawbacks of this pass

1. **[LOW][RELY]** Keepalive depends on the jsDelivr URL staying reachable. If pdfjs-dist removes that sample PDF, the cron warms but returns 4xx. Mitigation: function still boots, so warmth goal is met. **Fix later:** host a 1 KB stub PDF in Supabase Storage and point keepalive at it.
2. **[LOW][OBS]** No structured log line on `pdf-proxy` (`source, ms_to_first_byte, ms_to_ready, fallback_used`). Carry-over from r2. **Fix later:** add a single `console.log(JSON.stringify({…}))` at the end of each request branch — under 10 LOC.
3. **[LOW][UX]** `--app-breadcrumb-h` is scoped to documentElement, so any other page also reading it inherits MyCourseDetail's value while that route is mounted. Currently only one consumer, no conflict. **Fix:** if another page introduces its own breadcrumb, scope the var to a nearer parent.

## Wins
- Bundle gate runs on every `npm run build` (`postbuild`), so regressions are caught before APK.
- `pdf-proxy` warmth no longer depends on student traffic.
- Sticky-offset bug class permanently retired via measurement, not a magic number.
